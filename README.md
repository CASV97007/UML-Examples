# UML

## Diagrama de Casos de Uso

### Objetivo

El diagrama de casos de uso representa el comportamiento funcional de un sistema desde el punto de vista del usuario o de otros sistemas externos.

Su objetivo principal es mostrar:

- Qué funcionalidades ofrece el sistema.
- Quién interactúa con el sistema.
- Qué relación existe entre los usuarios externos y las funcionalidades.
- El alcance general del sistema.

No describe cómo se implementa internamente el sistema, sino qué hace el sistema desde una perspectiva funcional.

---

## ¿Qué es un caso de uso?

Un caso de uso representa una acción, funcionalidad o servicio que el sistema ofrece a un actor.

Ejemplos:

- Iniciar sesión
- Registrar usuario
- Crear pedido
- Consultar factura
- Realizar pago
- Descargar informe

Normalmente se escribe con un verbo en infinitivo o una frase de acción.

Ejemplo correcto:

```text
Registrar cliente
Realizar compra
Consultar historial
